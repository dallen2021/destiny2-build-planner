# Next.js Rebuild Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the broken legacy Express deploy surface with a Vercel-native Next.js foundation that can support Bungie auth, current API data, and read-only inventory.

**Architecture:** Build a clean Next.js App Router app at the repo root, with isolated TypeScript modules for Bungie OAuth, hash handling, session boundaries, and inventory normalization. Keep the first UI utilitarian and real, then layer optimizer behavior after the data foundation is tested.

**Tech Stack:** Next.js App Router, React, TypeScript, Vitest, ESLint, CSS modules/global CSS, native `fetch`, Node crypto, Vercel deployment.

```ts
const UINT32_SIZE = 0x100000000;
const INT32_MAX = 0x7fffffff;

export function toSignedHash(hash: number | string): number {
  const value = Number(hash);
  if (!Number.isFinite(value)) {
    throw new TypeError(`Invalid Destiny hash: ${hash}`);
  }
  return value > INT32_MAX ? value - UINT32_SIZE : value;
}

export function toUnsignedHash(hash: number | string): number {
  const value = Number(hash);
  if (!Number.isFinite(value)) {
    throw new TypeError(`Invalid Destiny hash: ${hash}`);
  }
  return value < 0 ? value + UINT32_SIZE : value;
}
```
